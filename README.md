# Задание 3 - Telegram-бот «Аналитик данных»

## Описание задания
Telegram-бот, который анализирует таблицы с помощью LLM-агента. Пользователь
присылает файл (CSV/TSV/Excel), при желании добавляет к нему инструкцию (на
что обратить внимание). В ответ бот возвращает текстовый отчёт с ключевыми
метриками, инсайдами и графиками.

Модель не получает готовые цифры в запросе и ничего не пересказывает. Ей дают
один инструмент - `run_python` - и она сама пишет Python-код, выполняет его по
таблице, читает вывод и продолжает, пока не соберёт отчёт. То есть это полноценный
агент с интерпретатором кода: каждое число и график в отчёте получены реально
выполненным кодом, а не выдуманы.

## Архитектура проекта и принцип работы
Проект разбит на четыре файла, каждый отвечает за свою часть.

### bot.py
Telegram-интерфейс на aiogram.
- принимает файл и инструкцию пользователя
- проверяет формат и размер файла
- прогоняет инструкцию через эвристику prompt-injection
- запускает анализ и отправляет отчёт с графиками обратно в чат

### agent.py
Цикл агента и общение с моделью через OpenRouter.
- собирает схему датасета (строки, колонки, типы, пропуски) - без значений ячеек
- в цикле даёт модели единственный инструмент `run_python`
- модель пишет код, бот его выполняет и возвращает модели вывод и ошибки
- модель итерирует и в конце выдаёт финальный отчёт

### sandbox.py
Запуск кода модели в отдельном процессе.
- ставит таймаут на выполнение и убивает зависший процесс
- на Linux ограничивает CPU и память (через `resource`)
- передаёт в процесс урезанное окружение - без токена бота и API-ключа

### sandbox_runner.py
Сам изолированный процесс, который реально выполняет код.
- грузит таблицу в `df` (подбирает разделитель и кодировку)
- выполняет код модели, ловит stdout, ошибки и графики
- возвращает результат родителю одной JSON-строкой

## Защита от prompt-injection
Защита от prompt-injection сделана в три слоя:

Данные отделены от команд - содержимое таблицы и текст пользователя считаются
данными, а не инструкциями. Значения ячеек в промпт вообще не попадают, имена
колонок экранируются, а системный промпт прямо требует игнорировать любые команды
внутри данных или инструкции.

Проверка на входе - перед запуском проверяется формат и размер файла, а текст
инструкции прогоняется через эвристику prompt-injection (RU/EN). При срабатывании
пользователь получает пометку, а модель - дополнительное предупреждение.

Изоляция выполнения - код модели выполняется не в процессе бота, а в отдельном
подпроцессе с таймаутом, лимитами CPU/памяти (на Linux) и без доступа к секретам.
Даже если защиту на уровне промпта обойдут, выполняемый код заперт.

## Инструкция запуска
Установить зависимости:
   ```
   pip install -r requirements.txt
   ```
Создать файл `.env` (пример `.env.example`) и вписать токен бота от @BotFather
и ключ OpenRouter (получить на https://openrouter.ai/keys):

   ```
   TELEGRAM_BOT_TOKEN=...
   OPENROUTER_API_KEY=sk-or-...
   MODEL=openai/gpt-oss-120b:free
   ```
Модель обязательно должна поддерживать вызов инструментов (function calling),
иначе агент не сможет запускать код. Подходящие модели - на
https://openrouter.ai/models с фильтром **Tools** (и **Free**, если нужна
бесплатная).

Запуск бота:

   ```
   python bot.py
   ```

## Работа бота
Написать боту `/start`, прислать файл (CSV / TSV / Excel) - бот посчитает метрики, построит графики
и пришлёт отчёт по разделам (обзор данных, ключевые метрики, инсайды, качество
данных, выводы).
